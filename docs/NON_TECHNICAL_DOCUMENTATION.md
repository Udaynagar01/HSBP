# HSBP Non-Technical Documentation

## 1) What is HSBP?
HSBP (Hyperlocal Service Booking Platform) is a web application where people can quickly book nearby home services like plumbing, electrical work, cleaning, carpentry, painting, and AC repair.

It acts like a bridge between:
- Customers (service seekers)
- Service providers (workers/business owners)
- Admin (platform manager)

---

## 2) Why this project was built
In day-to-day life, finding trusted local workers is often difficult and unorganized. HSBP solves this by offering one place to:
- Discover services
- Compare options
- Book easily
- Track request status
- Ensure provider verification through admin approval

---

## 3) Main users and what they can do

### Customer
- Sign up and log in
- Search services by category/name
- Book a service by giving address and time
- Track booking status
- Cancel pending bookings

### Service Provider
- Register as provider
- Get approved by admin
- Add and manage offered services
- Accept, reject, or complete bookings
- Monitor work requests

### Admin
- See all users and providers
- Approve/reject providers
- Monitor all bookings
- Reset passwords for accounts if required

---

## 4) End-to-end journey (simple story)
1. A customer creates an account
2. Customer browses available services
3. Customer books a selected service
4. Assigned provider receives booking request
5. Provider accepts and later marks completed
6. Admin oversees platform quality and approvals

If something goes wrong:
- Customer can cancel pending booking
- Admin can manage accounts and provider status

---

## 5) Key value delivered
- One platform for local home services
- Faster booking process
- Clear status tracking (pending, accepted, completed, cancelled)
- Better trust via provider approval
- Separate dashboards for each role
- Presentation-ready modern UI

## 6) What was built from setup to final stage

### Phase 1: Foundation setup
- Project initialized with frontend + backend
- Database connected

### Phase 2: Core platform
- Service listing and booking features developed
- Provider management and admin panel created
- Booking lifecycle integrated

### Phase 3: Improvements
- Home page and polished UI/UX added
- Sidebar-based role dashboard layout added
- Search functionality and filters improved
- Toast notifications introduced for cleaner experience

### Phase 4: Advanced account features
- Profile editing
- Forgot/reset password flow
- Admin password change controls
- Better error handling and role clarity

### Phase 5: Stabilization
- Runtime page errors fixed
- JSX/layout structure corrected
- Role access edge cases fixed
- Final cleanup for smooth demo execution

## 8) Limitations (current version)

- Reset link is demo-mode (shown directly instead of email integration)
- No rating/review system yet
- No production deployment pipeline included
